## 一、文档与代码一体化

**代码、结果和说明共存**

* 在同一个 Notebook 中可以同时包含：

  * 可执行代码
  * 运行结果（表格、图像、图表等）
  * 文本说明（Markdown）
  * 数学公式（LaTeX）
* 非常适合做“**可复现的分析过程**”，别人打开文件即可看到完整思路和结果。

---

## 二、基于 Cell 的交互式执行

**以 Cell（单元）为基本单位**

* Code Cell：编写并执行代码
* Markdown Cell：写说明、推导、结论
* 每个 Cell 可以**单独运行**，不必从头执行整个程序

**所见即所得**

* 代码执行后立即显示结果
* 支持 inline 显示图像、图表、HTML、视频等

---

## 三、强大的语言与内核支持

**多语言支持（通过 Kernel）**

* Python（最常用）
* R、Julia
* C++（xeus-cling）
* JavaScript、Scala 等

**内核与前端分离**

* Notebook 是前端
* Kernel 是执行引擎
* 支持远程计算（本地写代码，服务器跑计算）

---

## 四、非常适合数据分析与机器学习

**天然适配数据科学工作流**

* 与 NumPy / Pandas / Matplotlib / Seaborn / PyTorch / TensorFlow 深度集成
* 快速：

  * 读取数据
  * 试验特征
  * 调整模型
  * 可视化结果

**探索式编程（Exploratory Programming）**

* 边试边改
* 不需要事先写完整程序结构
* 非常适合原型验证和实验

---

## 五、可视化与富输出能力

* 支持：

  * 静态图（PNG、SVG）
  * 交互式图（Plotly、Bokeh）
  * HTML 表格
* 输出结果会被**直接保存**在 `.ipynb` 文件中

---

## 六、易分享与可复现性

**文件即成果**

* `.ipynb` 是一个 JSON 文件
* 可通过：

  * GitHub 直接在线预览
  * 导出为 HTML / PDF / Markdown
  * 作为实验报告或课程作业提交

**支持环境说明**

* 常配合 `requirements.txt`、`conda env` 使用
* 提高结果复现能力

---

## 七、在教学和演示中的优势

* 老师可以：

  * 一步步讲解
  * 现场运行代码
* 学生可以：

  * 修改参数
  * 立即看到变化
* 非常适合算法、数据结构、机器学习教学

---

## 八、局限性（必须客观看）

**不适合大型工程**

* 代码执行顺序依赖隐式状态
* 不利于模块化和长期维护

**容易产生“状态混乱”**

* Cell 可乱序执行
* 表面结果正确，但重启内核后无法复现

**版本控制体验一般**

* JSON 结构导致 Git diff 不友好

---

## 九、一句话总结

> **Jupyter Notebook 的核心价值在于：**
> 把「代码 + 思考过程 + 结果展示」放在同一个交互式文档里，极大提升探索、实验、教学和分析的效率。
