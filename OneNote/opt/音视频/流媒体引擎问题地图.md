## 一、时间与时钟体系（Timing / Clocks）

这一维是所有多媒体系统的根基，解决的是：**“所有东西到底按谁的节奏跑？”**

### 1.1 全局时钟与同步

- 问题内涵：
  - 播放/采集/编码/网络发送，各自可能有自己的时钟（采集卡 / 声卡 / 系统时钟 / 渲染循环）。
  - 需要决定谁是“主时钟”（Master Clock），其他模块向它对齐。
- 典型子问题：
  - 音画对齐（A/V Sync）
  - 不同设备/模块的时间戳对齐（采集、解码、渲染）

### 1.2 时钟抖动（Jitter）与漂移（Drift）

- 抖动：短时间内时间戳/间隔不稳定（比如网络包到达时间不均匀）。
- 漂移：长时间累积的误差（比如采集卡时钟每小时慢 10ms）。
- 典型子问题：
  - 采集卡时钟漂移，长时间直播音画一点点错开
  - 如何设计“时间平滑器”（如滑动窗口 + 回归 / PLL）来估计真实时间并纠偏

## 二、网络与传输控制（Networking / Transport）

解决的是：**“生成的数据如何在有限且不稳定的网络中可靠/高效地送出去？”**

### 2.1 动态比特率（Dynamic Bitrate）与拥塞控制

- 场景：渲染仍然 60fps，网络突然变差，发送堆积。
- 子问题：
  - 降低码率 / 分辨率 / 帧率？优先丢什么帧？
  - 丢非参考帧（P/B 帧） vs. 降编码质量 vs. 加大缓冲？
  - RTMP/SRT/QUIC 等协议中的拥塞反馈和速率控制机制如何接入引擎？

### 2.2 缓冲与延迟管理

- 如何在：
  - 不增加太多延迟（延迟敏感，比如直播互动）
  - 又能平滑网络抖动
  - 之间找到平衡？
- 子问题：
  - Jitter Buffer 设计（缓冲多长？动态缩/扩？）
  - 卡顿时是“停一帧画面”还是“跳帧追上来”？

## 三、计算与内存（CPU/GPU/Memory Pipeline）

这一维解决的是：**“数据怎么在 CPU / GPU / 内存之间高效流动？”**

### 3.1 GPU 与 CPU 的异步数据传输（Zero-Copy）

- 问题本质：
  - 摄像头采集在系统内存（CPU），渲染/特效在显存（GPU）。
  - 每帧 memcpy 会导致 CPU 占用爆炸、带宽吃满。
- 子问题：
  - 如何实现零拷贝/少拷贝？
    - 共享纹理（Shared Texture）
    - 直接使用 NV12 / DMA-BUF / DXGI Shared Surface 等机制
  - 如何设计统一的 Buffer/Frame 抽象，让一个缓冲能在多个模块间复用？

### 3.2 同步与异步（Pipeline Scheduling）

- CPU 与 GPU 各有队列，如何防止：
  - CPU 等 GPU（pipeline stall）
  - GPU 等 CPU（feed 不上来）
- 子问题：
  - 渲染循环 vs 编码线程 vs 网络发送线程的调度
  - 提前准备帧（prefetch）、多重缓冲（double/triple buffering）

## 四、音视频信号处理（A/V Processing）

这一维解决的是：**“在时间轴上对数据做怎样的变形/处理，同时保持主观质量？”**

### 4.1 音频重采样与爆音控制

- 场景：
  - 为了对齐视频（或某个时钟），微调音频播放速度。
  调得不平滑，会出现 “咔哒” / “爆音”。
- 子问题：
  - 选择重采样算法：
    - 简单：线性插值
    - 高级：带低通的重采样滤波器（FIR/IIR）、切比雪夫、Lanczos
  - 渐进式/平滑调速：
    - 避免突然大幅拉伸/压缩
    - 在时间上分散误差，让人耳“感受不到”

### 4.2 视频滤镜链（Filter Chain）与格式转换

- 问题本质：
  - 滤镜链是一个“图像处理管线”，不同滤镜顺序会改变画面和性能。
- 子问题：
  - 滤镜顺序：
    - 先抠图再磨皮 vs 先磨皮再抠图，结果和性能可能都不同。
  - 像素格式：
    - 不必要的 RGB ↔ YUV 来回转，性能开销巨大
    - 尽量让一条链在同一色彩空间 / 同一格式中完成
  - 多滤镜融合：
    - 能否把多个滤镜融合成一个 Shader Pass 减少读写？

## 五、系统集成与稳健性（System Integration / Robustness）

这一维解决的是：**“系统在真实世界的复杂环境中如何自适应、不崩、不漂、不累积误差？”**

### 5.1 多设备、多模块的时间/状态统一

- 采集卡、摄像头、麦克风、系统时钟、网络时间，它们都可能有自己的节奏。
- 子问题：
  - 时钟域之间的转换与校准
  - 热插拔 / 设备重连时如何平滑恢复（不导致时间轴炸裂）

### 5.2 长时间运行的累积误差与修正

- 特别是在 24h/7d 运行场景里：
  - 内部累积浮点误差
  - 时钟缓慢漂移
  - 缓冲区长度累计变化
- 典型手段：
  - 定期对齐到某个权威时钟
  - 使用回归/滤波估计偏差并渐进修正（不是硬重置）
