## 1. 2D 渲染对象

广义上讲，**2D 渲染对象**通常包括：

1. **文本 (Text)**  
   - 本质：矢量的字形轮廓 + 字体度量信息（字宽、字距、行距等）。  
   - 渲染时通常会：
     - 通过字体（TrueType/OTF 等）把文字成字形路径；
     - 再光栅化成像素，或缓存为纹理（glyph atlas）。  

2. **图像 (Image / Sprite / Texture)**  
   - 本质：一块二维像素数据，附带宽高、颜色格式等。  
   - 常见文件格式（离线存储格式）：  
     - PNG、JPEG、WebP、BMP、TIFF 等。  
   - 在 GPU 里：  
     - 通常是某种纹理格式（RGBA8, BGRA8, etc.），像素数据已被上传到显存。

3. **视频 (Video)** —— 是有的，只是通常不是“原生图元”，而是：  
   - **视频 = 随时间变化的图像**  
   - 渲染管线里，经常是这样处理：
     1. 用视频解码器（软解或硬解）把视频文件（H.264 / HEVC / VP9 等容器如 MP4、MKV、WebM）解码成当前时间点的一帧图像；
     2. 解码出的这一帧是一个像素缓冲区（YUV 或 RGB）；
     3. 把这一帧上传或绑定到 GPU，作为一张**纹理**；
     4. 再把这张纹理绘制到一个矩形（quad）上。  
   - 所以在 2D 渲染层面上，**视频就是“随时间更新的图像纹理”**。  
   - 在 UI/游戏/引擎的抽象中，可能会有专门的 `Video`/`MovieClip`/`Media` 对象，但底层最终还是通过纹理来渲染。

因此：  
- 2D 渲染对象从“类型”上可以有文本、静态图像、视频、形状（线、矩形、圆、路径等），甚至粒子等。  
- 从底层实现上看，多数都可以归结为：  
  - 要么是**矢量路径 + 填充/描边**；  
  - 要么是**纹理+几何平面**（包括视频帧）。

## 2. 2D 对象的“格式规范”

### 2.1 文件/资源层面的格式

- **文本**：  
  - 源文本：UTF-8 / UTF-16 编码的纯文本。  
  - 字体：TTF、OTF、WOFF、WOFF2 等。  
- **图像**（资源文件）：  
  - PNG、JPEG、WebP、SVG（矢量）、PSD、HDR 等。  
- **视频**（资源文件）：  
  - 容器：MP4、MOV、MKV、WebM 等；  
  - 编码：H.264、H.265/HEVC、AV1、VP9 等。  

### 2.2 渲染引擎/框架 API 层面的对象格式

不同框架有自己的对象模型，比如：

- Web（HTML/CSS/Canvas）：  
  - 文本：`<span>`, `<p>`, `fillText` 等 API；  
  - 图像：`<img>`，canvas 的 `drawImage`；  
  - 视频：`<video>` 元素 + `drawImage(video, ...)`。  

- 游戏引擎（Unity、Unreal、自研）：  
  - 通常有 `Sprite` / `Texture2D` / `Text` / `VideoPlayer` 之类的对象；  
  - 内部格式规范会定义：
    - 坐标系（左上角/左下角，NDC 等）；  
    - 颜色空间（sRGB/Linear）；  
    - 支持的纹理格式；  
    - 混合模式（alpha blend、additive 等）。

## 3. 3D 渲染对象是什么？格式规范有哪些？

### 3.1 3D 渲染对象的类型

常见 3D 场景中会有：

1. **网格 (Mesh/Geometry)**  
   - 顶点（位置 Position）  
   - 法线 Normal  
   - 纹理坐标 UV  
   - 切线 Tangent  
   - 顶点颜色 Color  
   - 骨骼权重 Skinning weights 等。  

2. **材质/着色器 (Material/Shader)**  
   - base color / albedo  
   - normal map  
   - metallic / roughness / specular  
   - emission 等。  

3. **纹理 (Texture)**  
   - 本质同 2D 图像；  
   - 类型更丰富：2D、3D、CubeMap、Array、Volume 等。  

4. **光源 (Light)**  
   - 点光、方向光、聚光等。  

5. **摄像机 (Camera)**  
   - 视椎体、投影矩阵、视图矩阵等。  

6. **动画 (Animation)**  
   - 骨骼动画（skeleton / skinning）  
   - 关键帧动画（keyframe）  
   - Morph target / blendshape。  

### 3.2 3D 模型/场景的文件格式规范

常见 3D 资源文件格式有：

- **glTF / GLB**（推荐现代标准）  
  - Khronos Group 制定，主打 “3D 的 JPEG”。  
  - 支持 PBR 材质、网格、节点层次结构、动画、蒙皮等。  
  - glTF（JSON+二进制/外部文件），GLB（单一二进制封装）。  

- **OBJ**  
  - 非常老牌、结构简单：顶点、法线、UV + MTL 材质文件。  
  - 不支持骨骼动画、复杂材质等。  

- **FBX**  
  - Autodesk 的格式，广泛用于 DCC 工具（Maya、3ds Max、Blender）。  
  - 支持网格、层次、动画、蒙皮等。  
  - 规范较复杂，多用官方 SDK。  

- 其他：  
  - COLLADA（.dae）、3DS、PLY、STL（多用于打印）、USD/USDZ（Pixar/Apple，影视&AR 领域越来越常见）。  

这些格式的“规范”，由相应组织/公司发布：
- 如 glTF 2.0 规范详细定义了：  
  - 节点、网格、访问器（accessor）、bufferView、材质、纹理、动画等数据结构。

## 4. 设计自己引擎/框架的“对象格式规范”

可以简单分层设计：

### 4.1 2D 层：

- 基础对象类型：
  - `Text`：内容、字体、字号、颜色、对齐、换行策略；  
  - `Image`：纹理引用、裁剪区域、UV、拉伸模式（9-patch 等）；  
  - `Video`：媒体源、当前时间、播放控制，内部持有一个“当前帧纹理”引用；  
  - `Shape`：矩形、、路径、描参数、填充参数；  
- 通用属性：
  - 坐标（position）、尺寸（size）、旋转、缩放、透明度、混合模式、z-order。  

### 4.2 3D 层：

- `Mesh`：  
  - 顶点缓冲、索引缓冲、子网格（submesh）列表。  
- `Material`：  
  - shader 名、参数（颜色、纹理、数值）。  
- `Node` / `Entity`：  
  - 变换矩阵（位置、旋转、缩放）  
  - 组件（MeshRenderer、Light、Camera、Animator 等）。  

文件格式可以自己定义 JSON/二进制，或者直接采用 glTF 等标准。
