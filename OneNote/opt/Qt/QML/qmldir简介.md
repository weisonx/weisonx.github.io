## 📌 Qt QML 核心笔记：彻底搞懂 qmldir 文件## 📄 什么是 qmldir？
qmldir 是 Qt QML 开发中用于组织、管理和定义自定义 QML 模块（Module）的纯文本配置文件。

* ⚠️ 致命细节：文件名必须全小写，且绝对不能有任何后缀名（如 .txt）。

------------------------------
## 🎯 什么时候必须用它？（核心应用场景）

* 优雅导入（告别相对路径）：避免写出 import "../../../components" 这种易碎的路径。配置后可直接 import MyModule 1.0。
* 跨项目复用（私有组件库）：将通用的 UI 控件（如自制图表、按钮）封装成独立模块，供多个不同的软件项目直接共享。
* C++ 混合编程（桥接插件）：当需要用 C++ 编写高性能逻辑或调用底层硬件时，通过 qmldir 关联编译出的 .dll / .so 动态库。
* 大项目解耦（模块化管理）：团队多人协作时，将大型项目按业务拆分成多个独立模块（如设置模块、主界面模块），降低代码耦合度。
* 多版本控制与代码提示：支持在同一个模块中并存 1.0 和 2.0 版本的组件；同时可通过关联 .qmltype 文件为 Qt Creator 提供完美的自动补全提示。

------------------------------
## 📝 常用语法速查

# 1. 声明模块的 URI 标识（必须放在首行，名称与文件夹同名）
module MyControls

# 2. 注册 QML 组件：[组件名] [版本号] [文件相对路径]
# 注意：组件名首字母必须大写！
CustomButton 1.0 CustomButton.qml
MySlider     2.0 components/MySlider.qml

# 3. 注册 JavaScript 脚本
MyScript 1.0 utils.js

# 4. 注册 C++ 插件（若有）
plugin mycpp_plugin_name

# 5. 关联类型提示文件（用于 IDE 代码高亮和补全）
typeinfo mymodule.qmltype

------------------------------
## 🚀 避坑指南（新手常犯错误）

   1. 大小写敏感：qmldir 中声明的组件名称必须与实际 .qml 文件同名，且首字母必须大写。
   2. 导入路径要填“父目录”：在 C++ 中使用 engine.addImportPath("path/to/dir") 时，该路径必须是存放模块文件夹的上一级目录，而不是模块文件夹内部。
   3. 缓存未刷新：如果将 qmldir 放入了 Qt 的 qrc 资源文件中，修改 qmldir 内容后，必须在 Qt Creator 中执行 “清除 (Clean) -> 重新构建 (Rebuild)”，否则修改不会生效。
