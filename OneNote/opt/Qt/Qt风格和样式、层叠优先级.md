# 调色板 `QPalette`
* 包含了部件各种状态的颜色组（激活、失效、非激活三种状态的颜色组）
* 调色板可以重写事件实现更多的颜色效果

# Qt样式表 `Qt Style Sheets`
* 除了那些能够通过子类化QStyle更改的外观，其余的都可以使用Qt样式表来美化
* 样式表不像调色板那样受不同平台的准则和本地主题引擎所限制，而可以执行所有的自定义操作
* 样式表应用于当前的部件风格之上

## 层叠：
* 样式表可以设置在QApplication 上、父部件上或子部件上。部件有效的样式表是通过部件祖先的样式表和QApplication上的样式合并得到。（即父部件的样式会被逐层继承，注意是部件之间的父子关系）
* 对于自定义类继承qt类样式规则：QSS 不会关心 C++ 的继承链，只关心 Qt 元对象系统提供的信息（meta-object）。继承自 QPushButton，因此 Qt 的样式引擎会认为它也是一个 QPushButton。Qt 样式系统只认继承自 QObject / QWidget 类的那条继承链。
* 当发生冲突时，部件自己的样式表优先于任何继承的样式表，同样，父部件的样式表优先于祖先的样式表。
* 层叠的一个特点是后设置的先生效，比如子先设置，但是父后设置，父还是会覆盖子原来的设置

## 样式规则
* 由选择器和声明组成，选择器指定了影响的部件，声明指定了所设置的属性
* 选择器有多种类型，如通用选择器、类型选择器、属性选择器等
* 其子控件如QComboBox的下拉按钮的样式也可以修改
* 伪状态设置样式：如鼠标悬停才应用该样式
* 不同类型选择器指定了相同的属性会产生冲突，有优先级作为解决原则
* 使用了Qt样式表后，部件并不会自动从父部件继承字体和颜色设置，而是使用系统颜色

## 选择器
| 选择器 | 示例 | 说明|
| -- | -- | -- |
| 通用选择器 | * | 匹配所有部件
| 类型选择器 | QPushButton | 匹配所有QPushButton实例和它的所有子类 |
| 属性选择器 | QPushButton[Flat="false"] | 匹配QPushButton的属性flat为false的实例 |
| 类选择器 | .QPushButton | 匹配所有QpushButton实例，但不包含它的子类 |
| ID 选择器 | QPushButton#okButton | 匹配所有QPushButton中以OKButton为对象名的实例 |
| 后代选择器 | QDialog QPushButton | 匹配所有QPushButton实例，它们必须是QDialog的子孙部件 |
| 孩子选择器 | QDialog > QPushButton | 匹配所有QPushButton实例，它们必须是QDialog的直接子部件 |

# 部件模型
每个部件都被看作拥有4个同心矩形的盒子，分别是内容、填衬、边框、边距：适合矩形、圆形等规则形状的部件

## 不规则形状的模型
部件遮罩（mask）等方式实现不规则窗体、透明窗体、毛玻璃等效果。
