## 1. QPainter (传统的即时模式渲染)
QPainter 是 Qt 最早期的绘图 API，属于即时模式（Immediate Mode）绘图。

* 核心逻辑：每当窗口需要重绘（触发 paintEvent）时，程序会按顺序执行绘图指令（画线、画圆、贴图）。
* 渲染设备：可以在 QWidget、QPixmap、QImage 甚至打印机上绘图。
* 优点：
* 直观简单：适合绘制简单的静态界面或自定义小部件。
   * 高兼容性：不依赖强大的显卡，纯 CPU 软渲染也能跑得很稳。
* 缺点：
* 性能瓶颈：由于每一帧都要重新计算所有像素，处理成千上万个独立对象时会非常吃力。
   * 缺乏交互管理：它只管画，不负责管理“这个圆圈是不是被点击了”。

## 2. Graphics View Framework (基于图元的管理框架)
为了解决大量图形对象的管理问题，Qt 推出了这一框架。它属于保留模式（Retained Mode）。

* 核心逻辑：它维护了一个“场景（Scene）”，场景里存放着各种“图元（Item）”。你只需要告诉框架图元的位置，框架会自动处理遮挡、刷新和碰撞检测。
* 三大组件：
* QGraphicsScene：容器，管理所有图元。
   * QGraphicsView：窗口，提供观察场景的窗口（支持缩放、旋转）。
   * QGraphicsItem：具体的物体（如矩形、图片、甚至是嵌入的按钮）。
* 优点：
* 强大的交互：原生支持拖拽、选中、碰撞检测。
   * 局部刷新：只重绘变动的区域，能够轻松处理数以万计的对象。
* 缺点：
* 内存开销：每个 Item 都是一个对象，内存占用比纯绘图大。
   * 底层限制：默认主要依赖 CPU 进行布局计算，虽然可以开启 OpenGL 加速，但本质上还是 2D 逻辑。

## 3. Qt Quick / Scene Graph (现代硬件加速渲染)
这是 Qt 5 引入的全新架构，专门为移动端和现代流畅 UI 设计，完全基于 GPU 加速。

* 核心逻辑：使用 QML 声明界面，后台将其转换为 Scene Graph (场景图)。它将绘图指令打包发送给显卡，利用 OpenGL/Vulkan/Direct3D 直接渲染。
* 工作流：
* 渲染线程：渲染过程在独立线程中进行，UI 动画即使在主线程忙碌时也能保持流畅。
   * 批量渲染：它会智能合并绘图节点，减少 GPU 的切换次数（Batching）。
* 优点：
* 极高性能：支持 60FPS 甚至更高帧率的复杂动画。
   * 开发高效：QML 像写 HTML/CSS 一样简单，支持粒子系统、着色器（Shader）。
* 缺点：
* 硬件依赖：需要显卡支持（尽管有软件回退方案，但性能会大打折扣）。
   * 黑盒化：自定义底层底层逻辑比 QPainter 复杂。

------------------------------
## 总结对比

| 特性 | QPainter | Graphics View | Qt Quick (Scene Graph) |
|---|---|---|---|
| 设计范式 | 即时模式 (画完即忘) | 保留模式 (对象管理) | 声明式 + GPU 优化 |
| 主要渲染器 | CPU (Raster) | CPU + 可选 GPU | GPU 强依赖 |
| 交互能力 | 无 (需手动计算坐标) | 极强 (自带坐标变换) | 极强 (触摸、动效优先) |
| 适用规模 | 简单、少量图形 | 海量、高交互图元 | 流畅 UI、动效、跨平台 App |
| 典型案例 | 自定义进度条、静态图表 | 电路图设计软件、地图编辑器 | 手机端 UI、车机大屏、多媒体应用 |
