## 复合类型的不可变（本身、成员是否可变）

### Java

- Java因没有像C++的const（内置普通类型final除外），因其变量可随意赋值，不可变靠约定；但是要求不可变的情形下如有违反则会报错
- Java复合类型的变量实际是某个对象的指针，所以不可变（final）要求不能将变量指向另一个对象
- 其内部成员不作要求，根据实际类型定义决定；如对容器变量进行增删改不属于改变指向的对象

### Rust

- rust变量不可以变是其变量所绑定的值不可变
- 对于复合类型，只有可变的变量才能改变其内部成员

### C++
- C++变量不可变就是值不可变，复合类型内所有成员（不管成员定义成什么类型）均不可变（通过变量操作时）
- 如果违反操作，编译器会报错：
``` cpp
class A {
public:
	int a;

	void c() {
		b = 1;
	}

private:
	int b;

};

int main()
{
  // TestDll();
	const A a;
	a.a = 1; // 报错
	a.c(); // 报错

  return 0;
}
```
