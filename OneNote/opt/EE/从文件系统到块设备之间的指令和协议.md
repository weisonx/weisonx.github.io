### 1. 内核接口层：逻辑指令的发起

当文件系统（如 ext4, XFS）需要读写数据时，它不会直接操作硬件，而是通过标准的内核接口提交请求。

*   **核心数据结构：`struct bio` (Block I/O)**
    文件系统将文件偏移量转换为**逻辑块地址（LBA）**，然后构建一个 `bio` 结构体。这个结构体包含了数据在内存中的位置（页缓存）、在磁盘上的逻辑块号、以及操作类型。
*   **关键函数/指令：`submit_bio()`**
    这是文件系统向块设备层提交 I/O 请求的核心函数。它标志着数据从“文件管理”阶段进入“设备管理”阶段。
*   **操作类型（Opcode）：**
    *   **READ / WRITE**：最基础的读写指令。
    *   **FLUSH**：强制将缓存写入设备（如 `fsync` 系统调用）。
    *   **DISCARD / TRIM**：通知 SSD 某些数据不再使用（用于垃圾回收）。

### 2. 通用块层：I/O 调度与合并

在请求到达驱动程序之前，内核的通用块层会对这些 `bio` 进行处理，以提高效率。

*   **电梯算法：**
    内核不会来一个请求就处理一个，而是将请求放入队列。通过电梯算法（如 Noop, CFQ, Deadline, BFQ, Kyber），内核会对请求进行**排序**和**合并**。
    *   *排序*：将磁头移动距离最近的请求放在一起处理（针对机械硬盘优化）。
    *   *合并*：如果两个请求是相邻的块，就合并成一个大请求。
*   **关键数据结构：`struct request`**
    `bio` 会被组装成 `request` 结构，然后被发送给具体的设备驱动程序。

### 3. 驱动与硬件协议层：物理指令的传输

这是“指令”和“协议”真正发挥作用的地方。驱动程序接收内核的 `request`，将其翻译成特定的硬件指令，通过总线协议发送给块设备控制器。

根据物理接口的不同，使用的协议完全不同：

#### A. 传统硬盘/SSD (SATA/SAS 接口)
*   **传输协议：** **ATAPI** (针对 SATA) 或 **SCSI** (针对 SAS/SATA)。
*   **指令集：**
    *   **SCSI 命令集**：即使在 SATA 硬盘上，操作系统通常也是通过模拟 SCSI 命令集（SAT - SCSI ATA Translation）来与之通信。
    *   **常见指令：**
        *   `READ(10)` / `READ(16)`：读取数据（括号内数字代表操作码字节长度）。
        *   `WRITE(10)` / `WRITE(16)`：写入数据。
        *   `SYNCHRONIZE CACHE(10)`：对应文件系统的 `FLUSH`，确保数据落盘。
        *   `UNMAP`：对应 SSD 的 TRIM 指令。

#### B. 高性能 NVMe SSD (PCIe 接口)
*   **传输协议：** **NVMe (Non-Volatile Memory Express)**。
    *   NVMe 是为闪存存储设计的，不再基于古老的 SCSI 指令集，而是基于**队列**和**内存寄存器**的直接操作。
*   **指令机制：**
    *   驱动程序将指令写入**提交队列**，硬件处理完后将状态写入**完成队列**。
*   **常见指令：**
    *   `NVM Read` / `NVM Write`：读写指令。
    *   `Compare`比较数据。
    *   `Write Zeroes`：快速写入零（无需传输实际数据）。
    *   `Dataset Management`：用于处理 TRIM/UNMAP。

#### C. 虚拟化环境
*   **传输协议：** **Virtio**。
*   **指令机制：**
    *   前端驱动在**Virtqueue**（虚拟队列）中放入描述符，通知后端模拟设备处理。
    *   常见指令类型：`VIRTIO_BLK_T_IN` (读), `VIRTIO_BLK_T_OUT` (写), `VIRTIO_BLK_T_FLUSH`。

### 总结：从文件到设备的映射图

| 层级 | 关键动作/指令 | 数据结构/协议 | 作用 |
| :--- | :--- | :--- | :--- |
| **文件系统** | `read()`, `write()` | Page Cache, Inode | 将文件路径转换为逻辑块号 (LBA)。 |
| **VFS/内核** | `submit_bio()` | `struct bio` | 封装 I/O 请求，向下传递。 |
| **通用块层** | I/O 调度 | `struct request`, Elevator | 合并、排序请求，优化性能。 |
| **驱动层** | `scsi_queuecommand`, `nvme_submit_cmd` | 驱动程序 | 将内核请求转换为硬件指令。 |
| **硬件协议** | SCSI CDB, NVMe Command | SATA, SAS, NVMe, PCIe | 物理传输指令，控制硬件动作。 |

**一句话总结：**
文件系统通过 `submit_bio` 提交基于 **LBA** 的读写请求，经过通用块层的调度后，设备驱动将其转换为 **SCSI 指令集**（针对 SATA/SAS）或 **NVMe 指令集**（针对 PCIe SSD），最终通过对应的总线协议完成数据传输。
