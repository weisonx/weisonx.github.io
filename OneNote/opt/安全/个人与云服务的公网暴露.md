# 公网暴露实例
### 核心含义
   
“公网暴露”意味着该实例不再受限于私有网络（VPC）的内部访问。互联网上的任何设备，只要知道其公网 IP 地址和开放的端口，理论上都可以尝试与其建立连接。 

合规暴露：如 Web 服务器需要对公网开放 80/443 端口，以便用户访问网站。

违规/风险暴露：如本应只在内网使用的数据库（3306 端口）或远程管理接口（22/3389 端口）无意中对全网开放，这极易引发暴力破解或漏洞攻击。 

### 工作原理
   
实现“公网暴露”通常涉及以下技术环节：

分配公网 IP：实例被分配了一个可以在全球互联网路由的唯一 IP 地址。

NAT/路由映射：在云网络架构中，实例通常位于私有网络内。通过 DNAT（目标地址转换） 或 公网网关，将发往公网 IP 的请求转发到该实例的私网 IP 上。

安全组/防火墙策略：这是决定是否“暴露”的关键。若安全组规则配置为 0.0.0.0/0（代表全互联网），则意味着该端口对全世界可见，完成了“暴露”。 

### 如何降低暴露风险

为了在不影响运维的情况下收敛暴露面，企业通常采用以下方案：

堡垒机/会话管理：通过加密隧道进行管理，无需在实例上监听公网入方向端口（零端口暴露）。

VPN 或 专线：将管理流量限制在加密的安全隧道内，不对公网开放。

负载均衡 (SLB/ALB)：通过负载均衡器作为唯一入口，后端实例隐藏在私网中，只暴露必要的业务端口。

云防火墙/WAF：在公网入口处进行流量清洗和入侵防御，过滤恶意扫描和攻击。 

# 个人PC

虽然普通家用电脑通常隐藏在路由器之后，但通过特定配置或软件，它也会像云服务器一样“暴露”在互联网中。 

以下是个人 PC 发生公网暴露的常见场景、原理及风险：

### 常见的暴露场景

主动开启远程桌面：为了在公司访问家里的电脑，用户在路由器上开启了 端口转发 (Port Forwarding) 或 DMZ（非军事区），将 3389 端口（Windows 远程桌面）直接映射到公网。

搭建个人服务：在电脑上运行了私有云盘（如 NAS、OwnCloud）、个人网站或游戏服务器（如我的世界 Minecraft），并配置了公网访问。

使用内网穿透工具：即便没有公网 IP，使用 FRP、NPS、花生壳 等工具也会通过中转服务器将本地端口暴露给公网用户。

拨号上网直接连猫：如果电脑直接连接宽带调制解调器（光猫）并进行 PPPoE 拨号，且未经过路由器的防火墙保护，电脑将获得一个独立的公网 IP，所有端口都直接面对互联网攻击。 

### 原理：从“隐身”到“现身”
   
通常情况下，路由器通过 NAT（网络地址转换） 技术充当“挡箭牌”，外界只能看到路由器的公网 IP，看不到你的 PC。 

暴露的本质：当你设置了端口转发，就相当于在路由器的防火墙上“打了一个洞”。当互联网上的任何人访问 公网IP:端口 时，路由器会不加过滤地将流量转发给你的个人 PC。 

### 个人 PC 暴露的风险
   
暴力破解：黑客会利用自动化脚本不停尝试你的远程桌面密码或服务登录密码。

系统漏洞攻击：如果你的操作系统没有及时更新补丁，黑客可以通过扫描发现开放端口，利用已知漏洞（如永恒之蓝）直接控制你的电脑。

隐私泄露：一旦被攻破，电脑里的照片、文档、账号密码都可能被窃取。 

### 如何检查与防护
   
检查方法：

查看路由器设置中是否有 “虚拟服务器”、“端口转发” 或 “DMZ” 开启。

在电脑浏览器访问 ip.skk.moe 或类似工具，查看显示的 IP 是否与路由器 WAN 口 IP 一致。

防护建议：

非必要不转发：不要随意将整个 PC 置于 DMZ 区。

使用 VPN/堡垒机：若需远程访问，建议通过 WireGuard、Tailscale 等加密隧道，而不是直接暴露端口。

强密码与二步验证：任何暴露的服务必须使用高强度密码。

# 端口转发和NAT区别

简单来说，NAT 是一项基础技术（门卫），而 端口转发是 NAT 的一种特定规则（特定包裹的转交）。

它们的关系是：端口转发（Port Forwarding）是建立在 NAT 技术之上的一个功能。

### NAT (网络地址转换) —— “隐身术”
   
NAT 的主要工作是将多个私网 IP（如你家里的手机、电脑）映射成一个公网 IP 去访问互联网。

工作方向：主要是从内到外。

作用：当你上网时，NAT 负责把你的请求发出去，并记住是谁发的。当数据回来时，NAT 再把数据准确地分发回你的电脑。

安全效果：它像一堵墙。默认情况下，互联网上的外部设备主动找你时，NAT 路由器会直接丢弃请求，因为它不知道该把这个请求给谁。这就是为什么你的个人 PC 默认是安全的。

### 端口转发 (Port Forwarding) —— “开后门”
   
端口转发是 NAT 里的一个静态配置项。它告诉路由器：如果有外部流量主动访问我的 公网IP:特定端口，请不要丢弃，而是直接转发给内网的 某台特定电脑。

工作方向：专门处理从外到内的主动请求。

作用：让外界能够绕过 NAT 的拦截，主动访问你内网里的某个服务（如远程桌面、个人网站）。

原理比喻：

NAT 就像一个大公司的总机，你（内网用户）给客户打电话，客户回电时，总机（NAT）能转接给你。

端口转发 就像你给总机设了一个分机号。你告诉总机：“只要有人打 8888 分机，别问是谁，直接接到我桌上。”

### 总结
NAT 是让大家都能上网，且默认把大家藏起来。

端口转发 是为了在 NAT 这堵墙上“打个洞”，让特定的流量（比如游戏服务器流量）能穿进来。
