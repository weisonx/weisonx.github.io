## 一、电梯 磁盘调度类算法

这一类问题本质上是：**一维空间上的请求调度**（楼层、磁道号），目标通常是减少等待时间、减少“来回跑”距离。电梯算法很多最早是从**磁盘调度算法**借来的。

### 1. 先来先服务（FCFS, First-Come First-Served）

- 场景：最简单的磁盘调度、电梯调度。
- 思路：  
  - 请求按到达顺序排队；  
  - 不考虑位置远近。
- 特点：公平、实现简单，但平均等待时间可能很差。

### 2. 最短寻道优先（SSTF, Shortest Seek Time First）

- 场景：磁盘读写头调度，或模拟电梯“离哪最近先服务”。
- 思路：  
  - 每次选距离当前所在位置最近的请求处理。  
- 优点：平均寻道时间短。  
- 缺点：可能导致“饥饿”——远处请求长期得不到服务。

### 3. 电梯算法 / SCAN 算法

- 场景：  
  - 典型是磁盘调度，也可直接用于电梯控制。  
- 思路（电梯类比）：
  1. 电梯先沿一个方向移动（比如向上），  
  2. 一路处理所有**沿途且同向**的请求，  
  3. 到达边界后再反向移动，  
  4. 重复以上过程。  
- 特点：  
  - 减少来回折返，有较好的“整体效率”；  
  - 相对较公平，等待时间比 SSTF 更均衡。

### 4. LOOK / C-SCAN / C-LOOK 等变种

- **LOOK**：  
  - 类似 SCAN，但不会一直走到最边界，而是走到这一个方向上最远的请求处就回头。  
- **C-SCAN（循环扫描）**：  
  - 一直往一个方向走（比如只向上），到顶后直接“瞬间”回到最底端（逻辑上跳回），然后再向上；  
  - 保证各个位置的平均等待时间更均匀。  
- **C-LOOK**：  
  - 结合 LOOK + C-SCAN，只在请求范围内循环。

## 二、CPU/任务调度算法

这些也是“调度算法”的经典应用，相当于“给谁先用 CPU”。

### 1. 先来先服务（FCFS）

- 场景：最简单的进程调度策略。  
- 思路：按到达顺序排队，先来先执行。  
- 缺点：长作业可能“压着”一堆短作业。

### 2. 短作业优先（SJF / SRTF）

- SJF（Shortest Job First）：非抢占式，优先执行估计执行时间最短的任务。
- SRTF（Shortest Remaining Time First）：抢占式，谁剩余时间最短谁先运行。
- 优点：平均等待时间最短。  
- 缺点：现实中很难准确知道作业时长；存在饥饿问题。

### 3. 优先级调度（Priority Scheduling）

- 思路：为任务设定优先级，高优先级先执行。
- 应用：实时系统、操作系统内核、高危任务优先处理等。
- 问题：低优先级任务可能长期得不到执行，需要**优先级老化**等机制防止饥饿。

### 4. 时间片轮转（Round Robin, RR）

- 场景：多用户操作系统、Web 服务、游戏服务器等。
- 思路：  
  - 给每个任务一个时间片（例如 10ms）；  
  - 时间片用完就换下一个任务；  
  - 像排队轮流使用 CPU。  
- 特点：响应时间较好，适合交互式系统。

## 三、缓存替换算法（页面置换、Web 缓存等）

**场景：** 内存管理（页缓存）、CPU cache、浏览器缓存、Redis/本地缓存等，当缓存满了，需要“踢掉谁”。

### 1. FIFO（先进先出）

- 思路：最早进入缓存的最先被淘汰。  
- 优点：实现简单。  
- 缺点：不考虑“是否常用”，性能不佳（Belady 现象）。

### 2. LRU（Least Recently Used）

- 思路：淘汰**最久未被使用**的数据。  
- 应用：  
  - 操作系统页表管理；  
  - Redis 的缓存淘汰策略之一；  
  - 各种本地缓存库。  
- 实现：链表 + 哈希表（O(1) 更新和访问）。

### 3. LFU（Least Frequently Used）

- 思路：淘汰**使用次最低**的数据。  
- 特点：更注重“热度”；实现稍复杂（需要统计频率）。

### 4. Random / Clock 等

- Random：随机踢一个，简单粗暴，有时效果还不错。  
- Clock：为实现近似 LRU 的一种高效页面置换算法，OS 常用。

## 四、网络 / 负载均衡 / 路由算法

### 1. 负载均衡算法

应用场景：多台服务器分担请求（Nginx、网关、RPC 框架）。

常见策略：

1. **轮询（Round Robin）**  
   - 每个请求依次分配给下一台服务器，简单且 fairly fair。
2. **加权轮询（Weighted RR）**  
   - 按权重分配，性能好的机器多分一点。
3. **最少连接数（Least Connections）**  
   - 优先把请求发给当前连接数最少的服务器。
4. **一致性哈希（Consistent Hashing）**  
   - 场景：分布式缓存（如分布式 Redis）、分库分表。  
   - 思路：按哈希值映射到“哈希环”上，减少节点增删时数据迁移量。

### 2. 路由与最短路径算法

- 场景：网络路由协议（OSPF、BGP 等）、导航软件、游戏寻路等。
- 典型算法：
  - **Dijkstra**：单源最短路径，适合非负权重；
  - **Bellman-Ford**：可处理负权边，可用于距离向量路由；
  - **A\***：在导航/游戏中加入“启发式函数”，搜索更快。

## 五、数据库 / 搜索引擎相关算法

### 1. 索引结构与查询优化

- B+ 树：  
  - 关系型数据库（MySQL/InnoDB）、文件系统索引常用。  
  - 用于区间查询、范围扫描。
- 哈希索引：  
  - 用于等值查询，O(1) 期望时间。
- 倒排索引：  
  - 搜索引擎、全文检索（ElasticSearch、Lucene）核心结构。

### 2. 查询执行与优化

- **连接算法**：Nested Loop Join Hash Join, Sort-Merge Join 等；
- **查询优化**：基于成本（Cost-Based Optimizer）选择最优执行计划。

## 六、操作系统其他经典算法

### 1. 内存管理 / 页面置换

- 与上面“缓存替换算法”类似：FIFO、LRU、Clock 等；
- 用于虚拟内存系统（页表 + 页面置换）。

### 2. 死锁检测与避免

- 银行家算法（Banker’s Algorithm）：  
  - 场景：资源分配，在保证安全状态下避免死锁。  
- 死锁检测算法：  
  - 定期检查资源分配图是否存在环。

### 3. 进程/线程同步

- 不严格叫“算法”，但有一批典型“模式”：
  - 生产者-消费者、读者-写者问题；  
  - 哲学家进餐问题。  
- 用到的工具：信号量、互斥锁、条件变量、Monitor 等。

## 七、控制 / 交通 / 工业类调度算法

### 1. 交通信号灯控制算法

- 固定周期控制：红绿灯固定时长循环；
- 感应式控制：根据车流量传感器动态调节；
- 自适应/智能控制：利用历史数据优化配时（有时用到强化学习、优化算法）。

### 2. 生产线调度 / 作业车间调度（Job Shop Scheduling）

- 场景：工厂生产计划，多工序、多机器。  
- 常见方法：
  - 贪心规则（如最短加工时间优先）；  
  - 启发式算法：遗传算法、模拟退火、Tabu 搜索等。

### 3. 电梯群控算法

- 多部电梯协调：  
  - 最近车优先、分区服务、预测客流（早高峰上行多、晚高峰下行多）；  
  - 部分采用智能调度（统计历史、机器学习）。

## 八、推荐 / 广告 / 排序中的算法

- 排序策略：  
  - 按“相关性”打分排序（学习排序：Learning to Rank）。  
- 推荐系统算法：  
  - 协同过滤（基于用户/物品的相似度）；  
  - 矩阵分解；  
  - 深度学习推荐模型（如 Wide & Deep、DeepFM 等）。  
- 广告竞价排序：  
  - GSP（广义二价拍卖）、eCPM = 出价 × CTR 等。
