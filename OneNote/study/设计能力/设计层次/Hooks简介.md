在编程领域中，Hooks（钩子） 是一种允许你“钩入”程序执行过程、系统事件或框架生命周期的技术。
它的核心逻辑是：在原本的程序执行路径中预留一个“挂钩点”，当程序运行到这个点时，会暂停并执行你挂载上去的代码，然后再继续（或者改变）原来的流程。
根据你所处的开发层面，Hooks 主要分为以下几类：
## 1. 前端框架中的 Hooks (React / Vue 3)
这是目前最火的概念。它将逻辑从笨重的“类”中解脱出来，变成了简洁的函数。 

* [React Hooks](https://zh-hans.react.dev/reference/react/hooks)：如 useState（管理状态）和 useEffect（处理副作用）。它们让你在不写 Class 的情况下，也能使用组件的状态和生命周期功能。
* Vue 3 Composition API：通常也被称为 Hooks。它通过 ref、computed 等函数，将原本分散在 data、methods 中的逻辑封装成可复用的“组合式函数”。 

## 2. 系统与底层开发中的 Hook (拦截技术)
这是一种“半路截胡”的技术，常用于 Windows 或 Android 开发。

* 原理：在系统消息（如鼠标点击、键盘输入）到达目标程序之前，先用 Hook 截获它。
* 用途：实现屏幕取词、按键记录、甚至游戏外挂或反作弊系统。 

## 3. 生命周期钩子 (Lifecycle Hooks)
许多框架都会在组件或应用的各个阶段提供钩子。

* 示例：onMounted（挂载后）、beforeDestroy（销毁前）。
* 形象理解：框架就像一条流水线，在特定的工位（阶段）留了挂钩，让你能在这个时刻做点你想做的事。 

## 4. 自动化工具中的 Hooks

* Git Hooks：在执行 git commit 或 push 动作前后触发。例如，你可以设置一个钩子，在提交代码前自动运行一遍代码格式检查。
* Webhooks：一种“反向 API”。当某个平台（如 GitHub）发生事件时，它会向你指定的 URL 发送一个请求（钩子），通知你的服务器进行处理。 [14, 15] 

一句话总结：Hooks 就是在“什么时候”发生“什么事”的连接点，它让原本封闭的程序流程变得可以被你灵活干预。
