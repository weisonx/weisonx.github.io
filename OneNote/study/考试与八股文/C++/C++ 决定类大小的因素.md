在 C++ 中，一个类（class）或结构体（struct）的大小主要受以下几个因素影响：

   1. 非静态数据成员：这是最基础的部分。类中所有非静态（non-static）成员变量的大小总和决定了基数。静态成员（static）不占用类实例的空间，因为它们存储在全局数据区。
   2. 内存对齐（Alignment）：编译器为了提高 CPU 访问效率，会在成员之间或类末尾填充字节（Padding）。通常类的大小必须是其最大成员大小的倍数。
   3. 虚函数（Virtual Functions）：如果类包含虚函数，编译器会为该类生成一个虚函数表（vtable），并在每个实例中插入一个虚函数表指针（vptr）。在 64 位系统上，这通常会增加 8 字节。
   4. 继承关系：
      * 普通继承：派生类的大小等于基类大小加上自身成员大小（同样考虑对齐）。
      * 虚继承（Virtual Inheritance）：为了解决菱形继承问题，虚继承会引入虚基类指针（vbptr）或类似的机制，增加额外开销。
   5. 空类（Empty Class）：为了确保不同的对象在内存中有唯一的地址，C++ 规定空类的大小为 1 字节。但如果该空类作为基类被继承，可能会触发空基类优化（EBO），此时它在子类中占 0 字节。
