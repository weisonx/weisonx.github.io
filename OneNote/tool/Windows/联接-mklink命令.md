将 C 盘的用户目录（C:\Users\你的用户名\.vscode）联接到其他盘，可以通过 Windows 自带的 MKLINK 命令来实现。这样可以让 VS Code 以为插件还在 C 盘，但实际上它们存储在 D 盘或 E 盘，从而彻底释放 C 盘空间。
请按照以下步骤操作（以迁移到 D 盘 为例）：
## ⚠️ 准备工作：彻底关闭 VS Code
在开始前，请务必完全关闭 VS Code 编辑器，确保没有后台进程在占用该文件夹。
------------------------------
## 🛠️ 核心操作步骤## 第一步：移动现有的 .vscode 文件夹

   1. 打开 C 盘路径：C:\Users\你的用户名。
   2. 找到 .vscode 文件夹，将其剪切（Ctrl + X）。
   3. 粘贴（Ctrl + V）到你想存放的新位置（例如 D:\ 盘根目录）。
   4. 此时，你的 D 盘应该有一个 D:\.vscode 文件夹，而 C 盘的用户目录下已经没有 .vscode 文件夹了。

## 第二步：以管理员身份打开命令提示符 (CMD)

   1. 点击 Windows 开始菜单。
   2. 搜索 cmd 或 命令提示符。
   3. 右键点击它，选择 “以管理员身份运行”（必须是管理员权限，否则会报错提示拒绝访问）。

## 第三步：执行联接命令
在打开的黑色 CMD 窗口中，复制并运行以下命令（注意：请将 你的用户名 修改为你电脑的实际用户名）：

`mklink /d "C:\Users\你的用户名\.vscode" "D:\.vscode"`

提示：如果你的新路径不是 D 盘根目录，请将 "D:\.vscode" 修改为你实际粘贴的目标路径（例如 "E:\Software\.vscode"）。
------------------------------
## ✅ 检查是否成功

   1. 返回 C 盘的 C:\Users\你的用户名 目录。
   2. 如果看到一个带有快捷方式小箭头的 .vscode 文件夹，说明联接成功。
   3. 双击这个文件夹，你应该能看到里面的 extensions 插件内容。虽然地址栏显示在 C 盘，但它其实是 D 盘的镜像。
   4. 重新打开 VS Code，检查你的插件是否都能正常加载和使用。
