Windows 应用程序的数据存放有明确的规范，主要依据 **Microsoft 的 Windows 应用程序认证要求** 和 **通用 Windows 平台 (UWP) / Win32 应用的最佳实践**。遵循这些规范可以确保应用在不同用户账户、不同 Windows 版本（如从 Win10 到 Win11）以及不同安装方式（如 MSI、商店、便携版）下都能稳定运行，并方便用户进行数据备份和清理。

以下是核心的存放规范，分为 **用户数据**、**程序配置** 和 **临时数据** 三大类：

### 1. 用户数据（User Data）

这是应用为用户创建、下载或保存的文件（如文档、项目文件、游戏存档、用户生成的图片等）。

-   **推荐路径：** `%USERPROFILE%` 下的标准库文件夹，或 `%LOCALAPPDATA%` / `%APPDATA%`。
-   **具体规范：**
    -   **文档类（如 .docx, .txt, .xlsx）：** 应存放在 `%USERPROFILE%\Documents`（即“我的文档”）。建议在文档内创建应用专属子文件夹，例如 `%USERPROFILE%\Documents\MyApp\`。
    -   **图片、视频、音乐类：** 分别存放在 `%USERPROFILE%\Pictures`、`%USERPROFILE%\Videos`、`%USERPROFILE%\Music` 下对应的子文件夹。
    -   **游戏存档或应用专属数据（不希望用户直接看到）：** 存放在 `%LOCALAPPDATA%\YourAppName\` 或 `%APPDATA%\YourAppName\`。**这是最常见的存放方式**，因为用户通常不会手动清理这些文件夹。
    -   **跨设备漫游数据（如设置、书签）：** 存放在 `%APPDATA%\YourAppName\`（对应 `C:\Users\<用户名>\AppData\Roaming`）。Roaming 目录下的数据会跟随用户在不同电脑间同步（如果域策略或微软账户同步开启）。

-   **禁止行为：**
    -   **绝对不要** 将用户数据存放在应用安装目录（如 `C:\Program Files\YourApp\`）下。因为普通用户没有写入权限，且卸载应用时会丢失用户数据。
    -   **避免** 直接放在 `C:\` 根目录或 `C:\Users\` 根目录下。

### 2. 程序配置与设置（Configuration & Settings）

这是应用自身的配置、缓存、日志等文件。

-   **推荐路径：** `%LOCALAPPDATA%`、`%APPDATA%` 或 `%PROGRAMDATA%`。
-   **具体规范：**
    -   **用户级配置（每个用户不同）：**
        -   **可漫游配置（如 UI 主题、窗口位置）：** `%APPDATA%\YourAppName\settings.ini` 或 `%APPDATA%\YourAppName\config.json`。
        -   **不可漫游配置（如缓存、本地数据库、日志）：** `%LOCALAPPDATA%\YourAppName\`。这是最推荐的配置存放位置，因为它不会占用网络同步带宽，且卸载时通常会被清理。
    -   **机器级配置（所有用户共享，如许可证、全局策略）：** `%PROGRAMDATA%\YourAppName\`（对应 `C:\ProgramData`）。注意：此目录默认是隐藏的，且需要管理员权限才能写入。
    -   **注册表：** 对于少量设置（如窗口大小、最近文件列表），可以使用 `HKEY_CURRENT_USER\Software\YourCompany\YourApp`。但 **不推荐** 存放大量数据或复杂结构，现代应用更倾向于使用 JSON/XML 文件。

-   **禁止行为：**
    -   **不要** 在 `%TEMP%` 中存放持久化配置（临时目录会被定期清理）。
    -   **不要** 在 `%USERPROFILE%` 根目录下创建大量隐藏文件夹（如 `.myapp`），这会干扰用户文件管理。

### 3. 临时数据（Temporary Data）

运行时产生的、不需要持久保存的文件（如解压缓存、渲染中间件、下载临时文件）。

-   **推荐路径：** `%TEMP%`（即 `C:\Users\<用户名>\AppData\Local\Temp`）。
-   **具体规范：**
    -   在 `%TEMP%` 下创建应用专属子文件夹，例如 `%TEMP%\YourAppName\`。
    -   **必须** 在应用退出时或下次启动时清理这些文件，避免占用磁盘空间。
    -   如果文件很大（>100MB），建议使用 `GetTempPath2` API 获取系统推荐的临时目录，或使用 `%LOCALAPPDATA%\Temp\YourAppName` 以避免影响系统临时目录。

-   **禁止行为：**
    -   **不要** 在 `%TEMP%` 中存放重要数据（可能被用户或系统清理工具删除）。
    -   **不要** 在 `%TEMP%` 中创建无意义的随机文件名（应使用 `GetTempFileName` 或 UUID 命名）。

### 4. 特殊情况：便携版应用（Portable Apps）

-   **规范：** 所有数据（配置、用户数据、缓存）都应存放在**应用自身所在目录**下的子文件夹中（例如 `.\Data\`、`.\Settings\`）。
-   **注意：** 便携版应用**不应**向 `%APPDATA%`、`%LOCALAPPDATA%` 或注册表写入任何数据，否则会破坏其“便携”特性。

### 5. 现代 UWP / MSIX 应用的特殊性

-   **沙盒化：** 应用被安装在 `C:\Program Files\WindowsApps\` 下的隐藏目录，普通用户无法直接访问。
-   **数据存放：** 系统会自动将数据映射到 `%LOCALAPPDATA%\Packages\<PackageFamilyName>\` 下，开发者通过 `Windows.Storage.ApplicationData` API 操作，无需手动指定路径。
-   **分类：**
    -   `LocalState`：本地数据（类似 `%LOCALAPPDATA%`）
    -   `RoamingState`：漫游数据（类似 `%APPDATA%`）
    -   `TempState`：临时数据（类似 `%TEMP%`）

### 总结：开发者应遵循的“黄金法则”

| 数据类型 | 推荐路径（环境变量） | 示例路径 | 说明 |
| :--- | :--- | :--- | :--- |
| **用户文档** | `%USERPROFILE%\Documents` | `C:\Users\Alice\Documents\MyApp\` | 用户直接创建/编辑的文件 |
| **用户配置（漫游）** | `%APPDATA%` | `C:\Users\Alice\AppData\Roaming\MyApp\` | 设置、书签、许可证 |
| **用户配置（本地）** | `%LOCALAPPDATA%` | `C:\Users\Alice\AppData\Local\MyApp\` | 缓存、日志、本地数据库 |
| **机器级配置** | `%PROGRAMDATA%` | `C:\ProgramData\MyApp\` | 全局策略、共享缓存 |
| **临时文件** | `%TEMP%` | `C:\Users\Alice\AppData\Local\Temp\MyApp\` | 运行时临时数据 |
| **安装目录** | `%PROGRAMFILES%` | `C:\Program Files\MyApp\` | **仅存放程序代码和不可变资源** |

**最佳实践建议：**

1.  **使用 `SHGetKnownFolderPath` API**（或 `Environment.GetFolderPath`）获取路径，而不是硬编码 `C:\Users\...`。
2.  **为应用创建专属子文件夹**（如 `YourCompany\YourApp`），避免与其他应用冲突。
3.  **在卸载时清理** `%LOCALAPPDATA%` 和 `%APPDATA%` 下的数据（通过安装程序或应用自身的卸载逻辑）。
4.  **避免在 `Program Files` 下写入任何文件**，除非是管理员手动操作。

遵循这些规范，你的应用将更符合 Windows 生态系统的预期，减少权限问题、数据丢失风险，并提升用户体验。
