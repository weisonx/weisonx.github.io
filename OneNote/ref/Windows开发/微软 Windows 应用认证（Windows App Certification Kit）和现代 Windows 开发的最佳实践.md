### 1. 安装与卸载规范

应用必须能够干净地安装和卸载，不能留下残留文件或注册表项。

-   **安装位置：** 默认安装到 `%ProgramFiles%` 或 `%ProgramFiles(x86)%`。用户安装时不应要求管理员权限（除非是系统级驱动或服务）。
-   **卸载清理：** 卸载时必须删除：
    -   安装目录下的所有文件（用户数据除外）。
    -   注册表中 `HKEY_LOCAL_MACHINE\SOFTWARE\YourCompany\YourApp` 和 `HKEY_CURRENT_USER\Software\YourCompany\YourApp` 下的键。
    -   `%APPDATA%`、`%LOCALAPPDATA%`、`%PROGRAMDATA%` 下应用创建的文件夹（**但应询问用户是否保留用户数据**）。
-   **使用标准安装程序：** 推荐使用 MSI、MSIX 或 WiX Toolset，避免使用简单的自解压包或批处理脚本。
-   **静默安装支持：** 应支持 `/quiet` 或 `/silent` 参数，便于企业批量部署。

---

### 2. 用户界面与用户体验规范

遵循 Fluent Design System，提供一致且现代化的交互体验。

-   **布局与控件：** 使用系统标准控件（如 Button、TextBox、ListView），不要自行绘制完全自定义的控件，除非有充分理由。
-   **高 DPI 支持：** 应用必须声明 DPI 感知（`PerMonitorV2`），确保在不同缩放比例下界面清晰、不模糊。使用 `SetProcessDpiAwarenessContext` 或清单文件声明。
-   **主题支持：** 应支持浅色/深色主题，并响应系统主题切换事件（`SystemEvents.UserPreferenceChanged` 或 `UISettings` API）。
-   **窗口行为：**
    -   窗口大小、位置、状态应保存并在下次启动时恢复（存储在 `%APPDATA%` 或 `%LOCALAPPDATA%` 的配置文件中）。
    -   不要创建无法关闭或隐藏的窗口（如无关闭按钮的托盘图标窗口）。
-   **触摸与笔输入：** 支持触摸手势（滑动、缩放）、笔压感（InkCanvas），并确保 UI 元素间距不小于 48px（触摸友好）。
-   **任务栏与托盘：**
    -   使用系统任务栏跳转列表（JumpList）提供常用操作。
    -   如需托盘图标，应使用 `Shell_NotifyIcon` 并遵循规范（右键菜单、左键默认操作、图标尺寸 16x16/32x32）。

---

### 3. 性能与资源管理规范

应用应快速启动、低内存占用、不消耗不必要的电量。

-   **启动时间：** 冷启动应在 5 秒内显示主窗口，10 秒内可交互。避免在启动时加载大量数据或执行网络请求。
-   **内存占用：** 空闲时应主动释放未使用的资源（如大图片、缓存数据）。避免内存泄漏（使用 `Performance Profiler` 检测）。
-   **CPU 使用：** 后台任务应限制 CPU 占用，避免持续占用超过 10%。使用 `ThreadPool` 或 `Task` 而非创建大量线程。
-   **电源管理：** 不要在后台执行高频率定时器或轮询操作，避免阻止系统进入睡眠状态（使用 `PowerCreateRequest` 仅在必要时声明）。
-   **磁盘 I/O：** 避免频繁写入小文件，使用批量写入或内存映射文件。临时文件应及时删除。

---

### 4. 安全与隐私规范

保护用户数据，遵循最小权限原则。

-   **权限声明：** 仅请求应用功能所需的最小权限。例如，不要请求“摄像头”权限如果只是文字编辑。
    -   Win32 应用：使用清单文件声明所需权限（如 `requestedExecutionLevel`）。
    -   UWP 应用：在 `Package.appxmanifest` 中声明功能。
-   **数据加密：** 敏感用户数据（密码、令牌、个人信息）必须加密存储。使用 `DataProtection API (DPAPI)` 或 `Windows.Security.Cryptography`。
-   **网络通信：** 默认使用 HTTPS，避免明文 HTTP。验证服务器证书，防止中间人攻击。
-   **代码签名：** 所有发布版本必须使用受信任的证书签名（EV 或标准代码签名证书），否则会被 SmartScreen 和 Windows Defender 拦截。
-   **输入验证：** 防止注入攻击（SQL、XSS、路径遍历），对所有用户输入进行验证和清理。
-   **沙盒隔离：** 优先考虑 MSIX 打包或 UWP 应用，利用其沙盒机制限制应用对系统和其他应用的访问。

---

### 5. 兼容性与部署规范

应用应在不同 Windows 版本、不同语言、不同硬件配置下正常工作。

-   **操作系统版本：** 明确声明最低支持版本（如 Windows 10 1809），并使用 `Version Helper API` 检测版本，避免调用不存在的 API。
-   **多语言支持：** 使用 `.mui` 文件或资源字符串，支持 Unicode。界面文字不应硬编码，应从资源文件加载。日期、货币、排序等应遵循系统区域设置。
-   **高对比度模式：** 应用应检测 `SystemParametersInfo(SPI_GETHIGHCONTRAST)` 并调整颜色，确保可读性。
-   **不同架构：** 提供 x86、x64、ARM64 版本（或使用 AnyCPU 编译的 .NET 应用）。避免在 64 位系统上强制使用 32 位路径。
-   **多显示器：** 窗口应在主显示器上打开，并正确处理不同 DPI 的显示器（PerMonitorV2 感知）。
-   **向后兼容：** 不要删除或修改公共 API/COM 接口，如果必须修改，应使用新版本号。

---

### 6. 辅助功能规范

使应用能被残障人士（视力、听力、行动不便）使用。

-   **键盘导航：** 所有交互元素必须可通过 Tab 键、箭头键、空格键、Enter 键访问。提供快捷键和加速键。
-   **屏幕阅读器支持：** 为 UI 元素设置正确的 `AutomationProperties.Name` 和 `AutomationProperties.HelpText`。使用 `UI Automation` 模式（如 InvokePattern、SelectionPattern）。
-   **高对比度与缩放：** 支持系统级高对比度主题，并允许用户通过 Ctrl+滚轮缩放界面（如浏览器）。
-   **颜色对比度：** 文本与背景的对比度至少为 4.5:1（WCAG AA 标准）。
-   **非文本内容：** 图片、图标应提供替代文本（alt text）。视频应提供字幕。

---

### 7. 后台任务与通知规范

后台任务应高效、不打扰用户，通知应遵循系统风格。

-   **后台任务：** 使用系统提供的后台任务触发器（如 `TimeTrigger`、`SystemEventTrigger`），避免常驻进程。后台任务应在 30 秒内完成（UWP）或合理时间内完成（Win32）。
-   **通知（Toast）：** 使用 `Windows.UI.Notifications` API 发送 Toast 通知，遵循通知模板，包含标题、正文、可选图片和按钮。不要滥用通知（如每 5 分钟一次）。
-   **通知分组：** 相关通知应分组，避免锁屏界面混乱。
-   **更新与进度：** 长时间操作（如下载）应在任务栏图标上显示进度条（`ITaskbarList3::SetProgressValue`）。

---

### 8. 错误处理与日志规范

应用应优雅地处理错误，并提供诊断信息。

-   **崩溃报告：** 注册 `SetUnhandledExceptionFilter`（Win32）或 `Application.UnhandledException`（.NET），收集崩溃信息并上传（需用户同意）。使用 Windows Error Reporting (WER) 集成。
-   **日志记录：** 使用结构化日志（如 ETW、EventLog 或文件日志），包含时间戳、级别（Info/Warning/Error）、上下文信息。日志文件存放在 `%LOCALAPPDATA%\YourApp\Logs`。
-   **用户反馈：** 错误信息应友好、可操作，不要显示技术堆栈（除非是调试模式）。提供“重试”、“取消”、“报告问题”等选项。
-   **状态保存：** 在意外关闭时（如断电），应自动保存用户工作状态，以便下次恢复。

---

### 9. 网络与通信规范

-   **连接性：** 检测网络状态（`NetworkInformation.GetInternetConnectionProfile`），在离线时提供本地缓存或提示。
-   **带宽友好：** 不要频繁轮询服务器，使用推送通知（WNS 或 WebSocket）。大文件下载应支持断点续传。
-   **代理支持：** 自动检测系统代理设置（`WebProxy.GetDefaultProxy`），不要硬编码代理地址。
-   **端口使用：** 避免使用知名端口（如 80、443）作为监听端口，应使用动态端口或注册到 Windows 防火墙。

---

### 10. 打包与分发规范

-   **MSIX 打包：** 强烈推荐使用 MSIX 格式打包 Win32 应用，以获得干净的安装、卸载、沙盒和更新体验。
-   **Microsoft Store 要求：** 如果上架商店，需满足：
    -   应用年龄分级（通过 IARC 问卷）。
    -   隐私政策链接。
    -   无恶意广告或诱导行为。
    -   支持 Windows 10/11 最新版本。
-   **更新机制：** 提供自动更新（如使用 Microsoft Store 更新、或内置更新检查），避免强制用户手动下载安装包。

---

### 总结：开发者应优先关注的规范清单

| 类别 | 核心要点 |
| :--- | :--- |
| **安装卸载** | 干净安装、完整卸载、MSIX 优先 |
| **UI/UX** | Fluent Design、高 DPI、深色主题、触摸友好 |
| **性能** | 快速启动、低内存、低 CPU、电源友好 |
| **安全** | 最小权限、加密存储、HTTPS、代码签名 |
| **兼容性** | 多版本、多语言、高对比度、多架构 |
| **辅助功能** | 键盘导航、屏幕阅读器、对比度、缩放 |
| **后台与通知** | 高效后台、规范通知、进度条 |
| **错误处理** | 崩溃报告、结构化日志、友好提示 |
| **网络** | 连接检测、带宽友好、代理支持 |
| **打包分发** | MSIX、商店要求、自动更新 |

遵循这些规范，你的应用将更容易通过 Windows 认证，获得更好的用户评价，并在企业环境中被接受。建议在开发周期中定期使用 **Windows App Certification Kit** 进行验证，确保符合标准。
