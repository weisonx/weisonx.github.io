在 C++23（以及现代 C++）中，static 和 inline 的作用已经不仅仅是字面上的“静态”和“内联”，它们更多地是在控制链接属性（Linkage）和内存生命周期。

## 1. static：控制“可见性”与“唯一性”## 修饰全局变量/函数（文件作用域）

* 作用：内部链接（Internal Linkage）。
* 效果：该变量或函数仅在当前 .cpp 文件内可见。即使其他文件定义了同名的 static 对象，也不会发生重定义冲突。

## 修饰局部变量（函数作用域）

* 作用：静态生命周期。
* 效果：变量在程序启动时初始化（或首次执行到此时），程序结束时销毁。它在函数调用之间保持数值不变。
* C++23 特性：多线程下初始化是线程安全的。

## 修饰类成员变量

* 作用：类共享。
* 效果：所有该类的对象共用同一个变量，不占用实例内存。
* C++23 技巧：现在可以配合 inline 在类内部直接初始化：static inline int count = 0;。

------------------------------
## 2. inline：解决“多重定义”的利器
在现代 C++ 中，inline 的主要作用不再是“建议编译器展开代码”，而是允许在多个翻译单元中定义同一个对象。
## 修饰函数

* 作用：ODR（One Definition Rule）放宽。
* 效果：允许函数定义在 .h 文件中被多个 .cpp 包含。链接器会自动处理，只保留一份最终副本，不会报“重定义”错误。
* 注意：编译器会根据复杂度自行决定是否展开，你控制不了。

## 修饰全局变量（C++17 引入，C++23 常用）

* 作用：Header-only 支持。
* 效果：你可以在头文件中直接定义全局变量：inline int global_val = 100;。
* 意义：以前必须在 .h 声明 extern，在 .cpp 定义。现在一个 inline 搞定，非常适合写“只有头文件的库”。

------------------------------
## 3. static inline：强强联手
当你看到 OBS 里的 static inline bool os_atomic_set_bool(...)，它的组合含义是：

   1. static：这个函数是当前文件私有的（Internal Linkage）。
   2. inline：如果我多次调用它，请尽量把它展开成指令，不要有函数压栈开销。

为什么不只用 inline？
如果只写 inline，它是外部链接。如果 OBS 的两个插件都定义了同名的 inline 函数但实现不同，链接器可能会随机挑一个，导致极其诡异的 Bug。加上 static 后，每个文件用自己的，绝对安全。
## 4. C++23 的新变化：static 也能修饰 operator()
在 C++23 中，Lambda 表达式和 operator() 可以被修饰为 static。

* 场景：static auto lambda = [](int x) { return x * x; };
* 好处：如果 Lambda 不捕获任何变量，将其设为 static 可以显式告诉编译器不需要生成 this 指针，进一步微操性能
