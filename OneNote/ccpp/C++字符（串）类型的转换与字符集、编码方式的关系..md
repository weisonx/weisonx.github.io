# 各种编程语言，其字符（串）类型，他们的区别是编码不一致，还是字符集不一致？

各种编程语言中的字符（串）类型的区别主要是编码不一致，而不是字符集不一致。

字符集是指一组字符的集合，例如ASCII字符集、Unicode字符集等。而编码则是将字符集中的字符映射到二进制数据的过程，例如ASCII编码、UTF-8编码等。

不同的编程语言中，字符（串）类型的实现方式和编码方式可能不同。例如，C语言中的字符类型是一个字节（8位），使用ASCII编码；而Java中的字符类型是两个字节（16位），使用Unicode编码。在C++中，字符类型的大小和编码方式取决于编译器的实现。

# 变量在内存中的数据内容

不管变量是什么类型，变量其在内存中的数据都是0、1字节序列；即不是ASCII字符也不是Unicode字符，这些都是编码后所显示的值；或者说是代表ASCII字符和Unicode字符的字节序列。

所以如果在编辑器显式将一个'a'（ASCII字符、ASCII编码）赋值给char a变量，那么'a'会将经过解码后的（ASCII字符的）字节序列存在内存（变量）中。

如果是Unicode字符如“中文”（此时编辑器使用的UTF-8编码或UTF-16编码），那么“中文”会将经过UTF-8解码或UTF-16解码后的（Unicode字符的）字节序列存在内存（变量）中。

对于C++而言，string、char*存的是字节序列，内部没有规定编码格式，所以字节序列在读写过程是没有任何转换而直接存储。而QString则不同，使用的是UTF-16编码，在读写字节序列时会进行一个转换后再保存。因此赋值给Qt的字节序列通常要告知其编码格式（当其不是UTF-16编码时）。

# 对于一个字节流数据，不同的字符串类型：

1. 通常在同一个操作系统，不同编程语言、不同字符串类型变量的直接传递，其字节流数据（内存）是不变的（相同的字节序以及相同的内存对齐方式的情况下）；

2. 通常在同一个操作系统，不同编程语言、不同字符串类型变量的进行转换后传递，其字节流数据（内存）通常会发生改变（除非其字符的字节流数据刚好在不同编码方式都是一致的，即兼容部分）；

举个例子：

UTF-8编码的例子：
```
char utf8[] = u8"你好，世界！";
for (int i = 0; i < sizeof(utf8); i++) {
    printf("%02x ", (unsigned char)utf8[i]);
}
输出结果为：
e4 bd a0 e5 a5 bd ef bc 8c e4 b8 96 e7 95 8c ef bc 81 

GB2312编码的例子：
char gb2312[] = "你好，世界！";
for (int i = 0; i < sizeof(gb2312); i++) {
    printf("%02x ", (unsigned char)gb2312[i]);
}
```
输出结果为：
`c4 e3 ba c3 a3 ac e4 b8 96 e7 95 8c 21`

转换规则：

> 1）. 如果UTF-8编码的字符是ASCII字符（即编码值小于等于0x7F），则直接将其转换为GB2312编码的字符。

> 2）. 如果UTF-8编码的字符是多字节字符，则需要进行转换。UTF-8编码的多字节字符的第一个字节的高位为1，后面的字节的高位为10。因此，可以通过判断第一个字节的高位来确定该字符的字节数。

> 3）. 对于UTF-8编码的多字节字符，需要将其转换为对应的Unicode字符，然后再将Unicode字符转换为GB2312编码的字符。这个过程需要使用Unicode字符集作为中间转换字符集。  
  
3. 不同编码方式的字符串需要转换是因为，该字符串类型就是代表其所存储的字符所用的编码方式，并基于此在打印/显示字符时使用相应的转换规则和字符集。所以字节流数据的编码方式最终要和字符串类型（编码方式）一致，这样才能正确解释。  

4. 若其编码方式的字符集不同，除了字符交集部分（大家都有的字符），转换是没有意义的，因为转换后始终不能正确解释出其原有的字符。除非两者定义一套转换规则（接口）使得其有意义。  

# 乱码

当解码一个字节序列时，如果其方式和编码时的方式不同，解码并打印出来的内容信息（字符集）就是乱码。
